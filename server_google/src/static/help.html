Чуть больше года у нас в компании используется методология Scrum для одного из проектов. Неотъемлемой частью скрама является доска. Обычно это пробковая доска на стене, разделённая по вертикали и по горизонтали на 3 полосы (по вертикали: не начатые – начатые – законченные, по горизонтали: высокая важность – нормальная важность – низкая важность). На доску вешаются задачи - стикеры, на которых указывается номер задачи, оценка в днях и что надо сделать. Чтобы обозначить кто взял какую задачу мы использовали закладки, которые наклеивали на задачи. Цвета стикеров обозначали тип задачи: пользовательская история, задача, баг, тест.

Обычная доска:

Из-за того что часть команды иногда вынуждена работать из дома возникла необходимость как-то сделать доску более доступной для распределённой команды. За 3 дня праздников в Ноябре 2010 появилась первая версия, которая до сих пор используется. Клиентская часть была сделана целиком на флеше и была призвана копировать look-and-feel обычной пробковой доски: любой мог зайти на сервер (аутентификация просто отсутствовала), повесить в любое место листок с задачей (drag&drop из палитры вверху доски), переместить листок на другое место или отредактировать его.

Электронная доска:

Небольшой ликбез по интерфейсу:

  • Двойной клик по note – открытие окна редактирования.
  • Чтобы добавить на доску новый note надо перетащить его (нужного цвета) из верхней панели на доску.
  • Чтобы добавить человека на note надо перетащить нужного человека из верхней панели на доску.
  • Перетаскивание в корзину (справа снизу) удаляет перетащенное (будь то note или человек).
  • Перетаскивание человека с одного note на другой note возможно.
  • На один note нельзя вешать больше 5 человек.
  • Хотите увидеть текущее состояние (с учётом чужих изменений) надо нажать кнопку Refresh сверху справа.
  • Слева от кнопки Refresh силуэт фотоаппарата – нажать для сохранения снимка доски в .png файл

Управление:

  • Если двигать мышью хватая за белое (там где нет note) то вместе с мышью движется доска
  • Если двигать мышью хватая за note, то note перемещается по доске (перемещения note сохраняются на сервере и видны всем).
  • Crtl+Scroll - зум (увеличение/приближение доски)
  • Перетащить note в корзину можно только через другой квадрат – только так срабатывает обработчик начала drag’а.
  • В режиме когда доска сильно уменьшена если с зажатым Ctrl навести на note, то он увеличивается до 100%

По новому серверу (на google apps engine). Возможности следующие:

  • Вход в систему с любым openid идентификатором
  • Любой человек вошедший в систему может создавать спринты
  • В любой созданный вами спринт можно добавлять других пользователей чтобы они тоже могли его редактировать (создавать/перемещать задачи итп)
  • Любой спринт можно сделать открытым (published), тогда он станет доступным только для просмотра всем (даже не вошедшим в систему)

В ролях:

  • Adobe Flash Builder (основная часть клиента)
  • Google Apps Engine + python + django (это вторая версия сервера, первая была на Apache + MySQL + PHP)
  • 6 дней от рассвета до заката (3 дня в ноябре 2010 и 3 дня в июне 2011) плюс исправления за весь период использования доски.
